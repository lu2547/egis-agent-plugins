# 图片布局规范

适用于图片与正文**并排放置**为容器块的页面布局规则。Strategist 和 Executor 在图片叙事意图为 *并排* 时共同遵循此规则。

**核心原则（并排布局）**：根据图片原始宽高比计算容器布局，确保图片完整显示——无多余留白、无裁切。

> **适用范围**：本规范仅适用于 *并排* 意图。其他意图（主视觉/全出血、氛围/背景、点缀/内联）使用全出血放置，比例对齐不作为约束且允许裁切——下方比例→分割表不适用。意图选择见 `references/strategist.md` §h。

---

## 布局决策流程

```
1. 确定叙事意图（主视觉 / 氛围 / 并排 / 点缀）——见 strategist.md §h
2. 若意图 = 并排：继续下方流程。否则：按叙事需要排版；本规范不适用。
3. 获取图片原始尺寸 → 计算比例 (width/height)
4. 根据比例选择布局类型
5. 计算图片最大显示尺寸
6. 分配剩余空间给文字区域
7. 将结果填入设计规范的图片资源列表
```

**执行时机**：若图片方案包含"B) 用户提供"，在 Strategist 八项确认之后、内容分析/大纲编写之前执行扫描并填充图片资源列表。

## 布局类型选择（并排意图）

| 图片比例        | 布局类型 | 图片位置 | 说明                              |
| --------------- | -------- | -------- | --------------------------------- |
| > 2.0（超宽）   | 上下分割 | 顶部全宽 | 图片横跨画布宽度，高度按比例      |
| 1.5-2.0（宽）   | 上下分割 | 顶部     | 图片宽度 = 内容区宽度，高度按比例 |
| 1.2-1.5（标准） | 左右分割 | 左侧     | 图片高度优先适配，宽度按比例      |
| 0.8-1.2（方形） | 左右分割 | 左侧     | 图片占满内容区高度，宽度按比例    |
| < 0.8（竖向）   | 左右分割 | 左侧     | 图片高度 = 内容区高度，宽度按比例 |

> 边界比例（如 1.5）：按文字量决定——文字多→左右分割；文字少→上下分割。

## 尺寸计算公式

### 画布参数（所有格式）

| 格式       | 画布      | 边距 (左右, 上下) | 内容区 (宽 x 高) | 标题高度 | 内容起始 Y |
| ---------- | --------- | ----------------- | ---------------- | -------- | ---------- |
| PPT 16:9   | 1280x720  | 60, 60            | 1160 x 600       | 60px     | 80px       |
| PPT 4:3    | 1024x768  | 50, 50            | 924 x 608        | 60px     | 70px       |
| 小红书     | 1242x1660 | 60, 80            | 1122 x 1500      | 80px     | 100px      |
| 微信朋友圈 | 1080x1080 | 60, 60            | 960 x 960        | 60px     | 80px       |
| Story      | 1080x1920 | 60, 120/180       | 960 x 1620       | 80px     | 140px      |
| 微信公众号 | 900x383   | 40, 40            | 820 x 303        | 40px     | 50px       |

> 下文中 **W** = 内容区宽度，**H** = 内容区高度（不含标题）。PPT 16:9 示例：W=1160, H=600。

### 上下布局计算

```
Image width = W = 1160 px
Image height = W / R = 1160 / R px
Text area height = H - image height - gap(20px)

验证: Text area height >= 150px（至少容纳 3-4 行文字）
若不满足 → 切换为左右布局
```

### 左右布局计算

**方法 1（高度优先，适合竖向图片）**：
```
Image height = H = 600 px
Image width = H x R = 600 x R px
Text area width = W - image width - gap(20px)
```

**方法 2（宽度约束，适合宽图转左右）**：
```
Image width = W x 0.7 = 812 px
Image height = image width / R
Text area width = W - image width - gap(20px)
```

**验证**：Text area width >= 280px；否则缩减图片区域宽度。

## 布局示例

### 超宽图片（比例 2.45）

```
原始: 1960x800, R=2.45 → 上下分割
图片: 1160x473, 文字区: 1160x147 → 7:3 上下
```

### 标准横向（比例 1.38）

```
原始: 1614x1171, R=1.38 → 左右分割
图片: 773x560 (左), 文字区: 367x560 (右) → 7:3 左右
```

### 宽图边界案例（比例 1.75）

```
原始: 1820x1040, R=1.75
尝试上下: 图片高度=663, 文字区=-43 ❌
切换左右: 图片 780x446 (左), 文字区 360x600 (右) → 7:3 左右
```

## 竖向画布覆盖规则

默认选择表假设**横向或方形画布**。竖向画布（高 > 宽）使用左右分割时两列都过窄——使用以下覆盖规则。

| 画布方向              | 图片比例          | 推荐布局 | 原因                     |
| --------------------- | ----------------- | -------- | ------------------------ |
| 竖向（小红书、Story） | > 1.5（宽）       | 上下分割 | 与横向画布一致           |
| 竖向（小红书、Story） | 1.2-1.5（标准）   | 上下分割 | 竖向画布上左右分割过窄   |
| 竖向（小红书、Story） | 0.8-1.2（方形）   | 上下分割 | 图片适合放在上半部分     |
| 竖向（小红书、Story） | 0.5-0.8（竖向）   | 左右分割 | 竖向图片在竖向画布上有效 |
| 竖向（小红书、Story） | < 0.5（极端竖向） | 左右分割 | 图片占一侧，文字占另一侧 |

> 方形画布（微信朋友圈 1:1）：使用标准横向规则。

## 多图布局

多图页面按以下公式均分内容区。

### 网格公式

```
columns = 列数
rows = 行数
gap = 20px (PPT 格式) 或 30px (社交格式)

cell_width  = (W - (columns - 1) * gap) / columns
cell_height = (H - (rows - 1) * gap) / rows
```

### 常见模式

| 图片数      | 布局    | 网格   | 说明                            |
| ----------- | ------- | ------ | ------------------------------- |
| 2（均横向） | 并排    | 2x1    | 两列等宽                        |
| 2（均竖向） | 堆叠    | 1x2    | 两行等高                        |
| 2（混合）   | 1大+1小 | 自定义 | 横向在上（全宽），竖向在右下    |
| 3           | 1大+2小 | 1+2    | 左侧大图（50%宽），右侧两张堆叠 |
| 4           | 网格    | 2x2    | 等大单元格                      |

### 示例：2x2 网格 PPT 16:9

```
W=1160, H=600, gap=20
cell_width  = (1160 - 20) / 2 = 570
cell_height = (600 - 20) / 2 = 290

图片位置:
  (60, 80)   570x290    (650, 80)  570x290
  (60, 390)  570x290    (650, 390) 570x290
```

> 多图页面：所有图片使用 `preserveAspectRatio="xMidYMid meet"` 确保单元格内显示一致。

## 禁止做法

| 禁止                  | 正确做法                                       |
| --------------------- | ---------------------------------------------- |
| 固定 50:50 或随意比例 | 根据图片比例动态计算                           |
| 强制宽图塞入方形容器  | 使用上下布局或增大图片区域宽度                 |
| 竖向图片放入窄横条    | 使用左右布局，图片在左                         |
| 图片留白超过 10%      | 重新计算布局或选择替代方案                     |
| 裁切关键图片内容      | 使用 `preserveAspectRatio="xMidYMid meet"`     |
| 文字区过小无法阅读    | 确保文字区 >= 150px（上下）或 >= 280px（左右） |

## 交接字段

本规范仅定义布局计算。将计算结果写入 [`svg-image-embedding.md`](svg-image-embedding.md) 中定义的图片资源列表：

| 字段          | 含义                               |
| ------------- | ---------------------------------- |
| `Ratio`       | 原始图片宽/高                      |
| `Layout plan` | 上下 / 左右 / 网格，含相关分割比例 |
| `Image area`  | 计算得出的显示矩形尺寸             |
| `Text area`   | 计算得出的剩余文字区尺寸           |

SVG `<image>` 语法、路径规则、`preserveAspectRatio`、外部引用、Base64 嵌入等详见 [`svg-image-embedding.md`](svg-image-embedding.md)。

### SVG 图片嵌入示例

完整显示（数据图表、并排——不可裁切）：

```xml
<image href="../images/xxx.png"
       x="60" y="80" width="780" height="446"
       preserveAspectRatio="xMidYMid meet"/>
```

裁切填充（仅限背景和主视觉图片）：

```xml
<image href="../images/bg.png"
       x="0" y="0" width="1280" height="720"
       preserveAspectRatio="xMidYMid slice"/>
```

## 自动化工具

```bash
python3 scripts/analyze_images.py <project_path>/images                    # 默认: PPT 16:9
python3 scripts/analyze_images.py <project_path>/images --canvas ppt43     # PPT 4:3
python3 scripts/analyze_images.py <project_path>/images --canvas xiaohongshu  # 小红书
```

`--canvas` 选择目标格式（默认 `ppt169`）。工具按上述公式计算布局类型（上下 / 左右）、图片显示区域和文字区域。输出为 Markdown 表格——可直接粘贴到图片资源列表。
