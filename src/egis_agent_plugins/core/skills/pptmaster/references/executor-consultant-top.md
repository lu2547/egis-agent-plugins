# 执行器 顶级咨询型 — MBB 级别顶级咨询风格

> 通用指南：executor-base.md。技术约束：shared-standards.md。

---

## 角色定义

MBB 级别（麦肯锡 / 贝恩 / BCG）顶级咨询风格 SVG 设计执行器。适用于战略规划、董事会汇报、投资路演、C-level 决策支持等**高端咨询**场景。核心特征：数据驱动洞察、金字塔结构叙事、极致简约。输出面向 C-level 决策者。

---

## SCQA 框架（叙事结构）

每个 MBB 级别演示文稿遵循 SCQA 叙事结构：

| 要素         | 作用                 | 典型页面         |
| ------------ | -------------------- | ---------------- |
| **S - 情境** | 建立共识背景         | 封面 / 第 1-2 页 |
| **C - 矛盾** | 引入问题/张力        | 第 2-3 页        |
| **Q - 疑问** | 隐含或显式的待解问题 | 过渡页           |
| **A - 答案** | 核心解决方案         | 所有正文页       |

**页面编排**：封面体现 S+C → 执行摘要回答 Q → 各章节展开 A → 结尾重访 S

**标题写法**：

| SCQA 要素 | 弱标题       | MBB 级别标题                           |
| --------- | ------------ | -------------------------------------- |
| S         | "行业背景"   | "数字化渗透率突破 60%，行业进入深水区" |
| C         | "面临的挑战" | "然而三重结构性矛盾制约规模化部署"     |
| Q         | "战略问题"   | "如何在 18 个月内从试点走向全面部署？" |
| A         | "解决方案"   | "三阶段路径：聚焦、扩展、规模化"       |

## 数据情境化（永远不孤立呈现数据）

> **黄金法则**：永远不要孤立展示单个数据点。每个数字都需要情境。

| 方法         | 模式               | 视觉实现                  |
| ------------ | ------------------ | ------------------------- |
| **时间对比** | "从 X 到 Y"        | 折线图 + 箭头标注变化幅度 |
| **基准对标** | "X vs 行业平均 Y"  | 柱形图 + 灰色虚线基线     |
| **竞争对比** | "我们 X vs 竞品 Y" | 并排柱形图，高亮自身数据  |
| **目标差距** | "实际 X / 目标 Y"  | 进度条 + 差距标注         |
| **排名**     | "M 中排第 N"       | 横向柱形图 + 高亮标记     |

**数据标注三要素**：每个数据可视化必须包含 (1) 数值本身（大号加粗） (2) 对比参考（基线/上期/竞品） (3) 含义解读（"So what?"）

```xml
<text x="160" y="280" text-anchor="middle" font-size="42" font-weight="bold" fill="#1E293B">97.3%</text>
<text x="160" y="310" text-anchor="middle" font-size="13" fill="#64748B">行业均值 82% | 竞品 A 89%</text>
<text x="160" y="335" text-anchor="middle" font-size="12" fill="#059669">领先行业 15.3 个百分点</text>
```

## 金字塔原理（结论先行）

> 高管不关心你的*过程*——他们关心*结果*。

```
         ┌─────────────────┐
         │    核心结论      │   ← 页面标题 / 要点提示
         └────────┬────────┘
      ┌───────────┼───────────┐
  ┌───┴───┐  ┌───┴───┐  ┌───┴───┐
  │ 论据1 │  │ 论据2 │  │ 论据3 │   ← 关键支撑点
  └───────┘  └───────┘  └───────┘
```

| 层级     | 位置       | 字号（示例 @ body=14px 密集基线） |
| -------- | ---------- | --------------------------------- |
| 核心结论 | 要点提示框 | 16-18px 加粗（~1.2x body）        |
| 论据     | 内容区正文 | 14-16px（~1x body）               |
| 支撑数据 | 图表/卡片  | 12-14px（~0.85x body）            |

> MBB deck 通常使用密集 `body` 基线（14-18px）以追求信息密度。上述 px 范围基于此假设；若特定咨询 deck 在 `spec_lock.md` 中声明了不同的 body 字号，使用上述比率而非字面 px 值。

**对比**：标题"市场调研结果"（错误）→"元宇宙需 5-10 年才能达到规模"（正确）；正文"我们调研了……发现……"（错误）→"三个论据：(1)… (2)… (3)…"（正确）

## MBB 级别布局模式

### 内容页标准结构（1280x720）

```
┌──────────────────────────────────┐
│ 渐变顶条 (0,0 → 1280,6)         │
├──────────────────────────────────┤
│ 断言标题 (x=40, y=50, 24px)     │
├──────────────────────────────────┤
│ 要点提示框 (x=40, y=75,         │  ← 深色背景 + 白色文字，一句话结论
│   w=1200, h=45)                  │
├──────────────────────────────────┤
│ 内容区 (x=40, y=140,            │  ← 图表 / 数据 / 分析
│   w=1200, h=520)                 │
├──────────────────────────────────┤
│ 来源 | CONFIDENTIAL | 页码       │  ← y=700, 10px
└──────────────────────────────────┘
```

### 战略路线图

三阶段水平排列，`<rect rx="8">` + 标题 + 行动列表，`<polygon>` 箭头连接：

```
聚焦核心 (0-6月)  ──>  扩展能力 (6-12月)  ──>  规模化 (12-18月)
x=40,w=380              x=450,w=380              x=860,w=380
```

### 对标矩阵

水平表格；自身行用主题色高亮，其余灰色。领先指标标绿，落后标红。用 `<circle>` 或 `<rect>` 构建评分点。

### 瀑布图（变化归因）

起点 → 增减因子 → 终点。正向因子绿色柱，负向因子红色柱，起止点深色柱，累计线用虚线连接器。

> 当 `page_rhythm = breathing` 时，MBB 适用的形式为**负空间驱动**：近乎空白画布上的单一要点陈述，或一张主导图表配一句战略含义（标准图表+洞察布局的不对称 2:8 退化）。MBB 的极致简约美学正*适合*这种形式——营销风格 deck 中的填充图像和装饰叠加不属于此。通用节奏纪律见 `executor-base.md §2.1`。

## 色彩的战略使用

MBB 级别的色彩使用极度克制——颜色服务于信息：

| 目的             | 方法                                               |
| ---------------- | -------------------------------------------------- |
| **聚焦**         | 目标数据用主题色，其余灰色                         |
| **降低认知负荷** | 同系列用不透明度区分（`fill-opacity` 1.0/0.6/0.3） |
| **语义**         | 绿=正面，红=负面，灰=基线                          |
| **品牌**         | 渐变顶条/装饰线用品牌色                            |

**戒律**：主色不超过 3 种；强调色全局最多用 2-3 处；数据系列用同色深浅变化，不用不同颜色；背景白色或极浅灰。

## 图表 vs 表格选择矩阵

| 场景            | 推荐形式     | 原因                      |
| --------------- | ------------ | ------------------------- |
| 对比 2-7 个类别 | **柱形图**   | 视觉对比直观              |
| 时间趋势        | **折线图**   | 时间序列清晰              |
| 精确值/大排名   | **表格**     | 50 个数据点用柱形图会混乱 |
| 比例构成        | **环形图**   | 比饼图更现代              |
| 二维定位        | **2×2 矩阵** | 战略象限分析              |
| 变化归因        | **瀑布图**   | 因子分解                  |

## 演讲备注风格

### 叙述基调

面向**高管/决策者**。金字塔结构——每句话都承载信息。语调沉稳、权威、有洞见。

### 备注撰写指南

备注为纯口述叙述（TTS）。不用方括号标记、不写 `要点：` / `时长：` / `Flex:` 行——见 [executor-base.md §8](executor-base.md#8-演讲备注生成框架)。

- **金字塔结构**：结论 → 论据 → 细节，以流畅行文写就。
- **数据情境化**：每个数字在同一句中配对比参考（"百分之二十三——几乎是行业均值百分之十二的两倍"）。
- **高管用语**："战略含义是……"、"核心洞察在于……"。
- **自然过渡**：第一页后的每页以衔接上一页的句子开头。

### 备注示例

```markdown
# 03_战略路径

在厘清了问题和机遇之后，我们来到最关键的部分——应对路径。建议采用三阶段策略：聚焦、扩展、规模化。第一阶段，未来六个月，集中资源攻克华东市场，该区域贡献了近三分之二的行业增长，而我们的渗透率仅为竞品 A 的一半。第二阶段向华南和华北复制模板，第三阶段借助数字化平台在十八个月的跨度内实现全国覆盖。
```

## MBB 级别质量检查补充

### 内容层面

- [ ] **SCQA 完整**：整体遵循情境-矛盾-疑问-答案结构
- [ ] **数据情境化**：每个数据点有对比参考
- [ ] **结论先行**：每页的要点提示框有一句话结论
- [ ] **MECE 原则**：分类分析互斥且穷尽

### 视觉层面

- [ ] **色彩有目的**：颜色服务于信息传达
- [ ] **留白充足**：不拥挤；信息可以"呼吸"
- [ ] **视觉层级**：结论 > 论据 > 细节，用字号/字重区分
- [ ] **品牌一致**：渐变顶条和页脚全篇统一

### 备注层面

- [ ] **金字塔结构**：每页结论先行
- [ ] **纯口述行文**：不含方括号阶段标记、不含 `要点：` / `时长：` / `Flex:` 元信息行（TTS 会逐字朗读）
- [ ] **数据行文情境化**：每个数字在同一句中配对比参考
