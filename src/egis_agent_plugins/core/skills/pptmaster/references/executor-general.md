# 执行器 通用视觉型 — 创意多元风格

> 通用指南：executor-base.md。技术约束：shared-standards.md。

---

## 角色定义

创意多元风格的 SVG 设计执行器。适用于产品介绍、培训材料、方案演示、营销推广等**非咨询类**场景。强调视觉冲击力和信息吸引力，在专业性与亲和力之间取得平衡。

---

## 通用型专属布局技法

### 1. 灵活多变的版式

通用风格不局限于固定模板；可根据内容自由选择版式：

| 版式                 | 适用场景           | 布局细节（1280x720）                          |
| -------------------- | ------------------ | --------------------------------------------- |
| 全图背景 + 文字叠加  | 封面、情感页       | `<image>` 填满画布 + 半透明叠加 + 居中标题    |
| 左右分割（图文混排） | 功能介绍、对比     | 左 x=40,w=580 / 右 x=660,w=580                |
| 三栏卡片             | 功能列表、团队介绍 | x=40,450,860 每个 w=380，等高卡片             |
| 上下分割             | 时间线、流程图     | 上区：标题+描述 h=250 / 下区：图表+内容 h=420 |
| 中心放射             | 核心概念、生态图   | 中心元素 + 4-6 个周围节点，线指向中心         |
| 瀑布 / Z 字型        | 叙事、案例展示     | 内容块左右交替，引导视线呈 Z 字形             |

### 2. 视觉节奏控制

- **信息密度交替**：数据密集页后跟"呼吸页"（大图/引言/过渡），防止观众疲劳
- **视觉重量平衡**：深色/大面积元素为"重"，浅色/小元素为"轻"——平衡左右/上下
- **重复与变化**：章节内保持布局一致性；章节间变化以保持新鲜感

### 3. 装饰元素使用

| 元素         | 用法               | 注意事项                                                                        |
| ------------ | ------------------ | ------------------------------------------------------------------------------- |
| 渐变块       | 背景分区、标题衬底 | 用 `<linearGradient>` / `<radialGradient>`，限 2-3 色                           |
| 圆角矩形卡片 | 内容容器、功能模块 | `rx="12"`；仅当卡片悬浮在照片/有色面板上时加轻阴影（见 shared-standards.md §6） |
| 图标点缀     | 列表前缀、功能标记 | 使用 `data-icon` 占位，尺寸 32-48px                                             |
| 编号圆       | 步骤流程、排名列表 | `<circle>` + 居中 `<text>`，主题色填充                                          |
| 分割线       | 内容分隔           | `<line>` 或 `<rect height="2">`，opacity 0.2-0.3                                |

## 视觉策略

### 色彩运用

- **大胆使用主题色**：封面和章节页可用大面积主题色背景
- **渐变增强层次**：标题条和卡片背景可用同色系渐变
- **对比制造焦点**：关键数字/词用强调色，与周围形成对比
- **色彩情绪匹配**：冷色调营造科技感，暖色调传递活力，深色调彰显稳重

### 图片处理策略

| 场景       | 策略                    | SVG 实现                                          |
| ---------- | ----------------------- | ------------------------------------------------- |
| 全屏背景   | 图片填满 + 暗色渐变叠加 | `preserveAspectRatio="xMidYMid slice"` + 渐变矩形 |
| 人物图展示 | 置于左/右，保持原始比率 | 控制宽度，高度自适应                              |
| 多图网格   | 网格排列，统一尺寸      | 等宽等高 `<image>` 矩阵                           |
| 人物照片   | 圆形裁剪效果            | `<circle>` 背景 + 方形图片叠加（后处理裁剪）      |

### 字体层级

字号遵循 `spec_lock.md` 中 deck `body` 基线为锚的梯度——以下 px 值为 **body ≈ 18px 时的示例**（其他基线乘以比率列：0.7x 注释 / 1x 正文 / 1.2x 副标题 / 1.6x 标题）。

```
标题层   → ~1.5-2x body  （如 28-36px @ body=18）  加粗，主题色或白色
副标题层 → ~1.2x body    （如 20-24px @ body=18）  中等字重，辅色
正文层   → 1x body       （如 16-18px @ body=18）  常规，深灰
注释层   → ~0.7-0.8x     （如 12-14px @ body=18）  浅灰，底部对齐
```

## 演讲备注风格

### 叙述基调

通用风格的演讲备注使用**对话式叙述**——如同与观众交谈，而非宣读报告。语调自然有节奏，适当使用修辞手法。

### 备注撰写指南

备注为纯口述叙述（TTS）。不用方括号标记、不写 `要点：` / `时长：` 行——见 [executor-base.md §8](executor-base.md#8-演讲备注生成框架)。

- **讲故事**：每页叙述采用"场景-冲突-解决"弧线。
- **用比喻**：让抽象概念具象化（"就像给系统装了涡轮增压器"）。
- **制造悬念**：在恰当时刻抛出问题，下一页解答——写成自然的反问句，不用 `[互动]` 标签。
- **口语化数据**：30% → "近三分之一"，2.5倍 → "翻了一倍多"。口语形式更自然时将百分比拼写为文字。
- **自然过渡**：第一页后的每页以衔接上一页的句子开头。

### 备注示例

```markdown
# 03_核心优势

聊完了市场格局，你可能会想——我们的机会在哪里？我们的核心优势可以归结为三个字：快、准、省。快，是因为部署时间从三个月缩短到了两周。准，是因为识别准确率达到了百分之九十七点三，远超行业平均的百分之八十二。省，是因为综合成本降低了近三分之一——这样的组合，如果你是决策者，很难不动心。
```

## 自检补充（通用型专属）

- [ ] 视觉节奏合理：数据密集页与呼吸页交替
- [ ] 装饰元素适度：服务内容，不喧宾夺主
- [ ] 图文比例适当：不全是文字墙，有视觉亮点
- [ ] 备注对话感强：读起来像说话，不像念稿
- [ ] 备注不含方括号阶段标记和 `要点：` / `时长：` 元信息行（TTS 会逐字朗读）
