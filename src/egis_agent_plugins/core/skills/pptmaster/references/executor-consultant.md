# 执行器 通用咨询型 — 咨询风格

> 通用指南：executor-base.md。技术约束：shared-standards.md。

---

## 角色定义

数据驱动的咨询风格 SVG 设计执行器。适用于商业分析报告、市场调研、运营复盘、战略建议等**通用咨询**场景。强调结构化信息呈现和数据可视化；风格简洁、清晰、专业。

## 咨询型专属数据可视化技法

### 1. KPI 仪表板设计

KPI 卡片是咨询报告中最常见的元素。标准布局（1280x720）：

```
4 卡片布局：每卡 280x180，间距 30
  卡片 1：x=45,  y=160
  卡片 2：x=355, y=160
  卡片 3：x=665, y=160
  卡片 4：x=975, y=160
```

**卡片内部结构**（从上到下）：

| 区域     | 内容           | 字号（示例 @ body=18px）          | 样式              |
| -------- | -------------- | --------------------------------- | ----------------- |
| 图标行   | data-icon 图标 | 32x32                             | 主题色            |
| 指标名   | "月活跃用户"   | 14px（~0.8x body）                | 灰色 #64748B      |
| 核心数字 | "120万"        | 36-42px（~2x body，核心数字区间） | 加粗，深色        |
| 趋势注释 | "+12.3% 环比"  | 12px（~0.65x body，注释区间）     | 绿=上升 / 红=下降 |

> px 值假设典型的密集咨询基线（`body` ≈ 18px）。若 deck 在 `spec_lock.md` 中声明了不同的 `body`，使用括号中的比率而非直接复制 px 值。

**趋势箭头约定**：
- 上升：`arrow-trend-up` 图标 + 绿色文字
- 下降：`arrow-trend-down` 图标 + 红色文字
- 持平：水平线图标 + 灰色文字

### 2. 图表配色约定

咨询风格图表使用**单色深度渐变**而非彩虹色：

```
主系列：   主题色 100% 不透明度
对比系列： 主题色 60% 不透明度
基线：     #94A3B8（灰色虚线）
高亮：     强调色（仅用于关键数据点）
```

### 3. 数据标注原则

- **直接数据标签**：将数值置于柱形图顶部——无需图例
- **标注趋势线**：在关键拐点添加文字注释（"政策变更"、"产品上线"）
- **对比基线**：用灰色虚线标注行业平均/目标值
- **单位一致**：同一图表内保持相同的数字单位和精度

### 4. 表格设计

咨询报告频繁使用表格展示精确数据：

| 设计元素 | 规范                                     |
| -------- | ---------------------------------------- |
| 表头     | 深色背景 + 白色文字，font-weight="bold"  |
| 斑马纹   | 隔行用 `fill-opacity="0.05"` 浅色背景    |
| 数字对齐 | 数字右对齐，文字左对齐                   |
| 高亮行   | 用强调色 `fill-opacity="0.1"` 标记关键行 |
| 边框     | 仅用水平线分隔行；避免完整网格线         |

## 咨询型专属布局模式

### MECE 分解树

主干在左，分支向右展开——相互独立、完全穷尽：

```
主指标 ──┬── 分支 A（45%）
          ├── 分支 B（30%）
          ├── 分支 C（20%）
          └── 分支 D（5%）
```

- 用 `<line>` 连接主干和分支
- 每个分支有 `<rect>` 容器 + 百分比标注
- 总和必须 = 100%（或明确标注"其他"）

### 驱动因素树

顶层指标 → 分解因子 → 行动。适合绩效归因分析：

```
营收增长 15%
  ├── 客单价 +8%   → 高端产品线扩展
  ├── 客户数 +5%   → 新渠道获客
  └── 复购率 +2%   → 会员体系优化
```

### 左图右文（图表解读）

核心咨询报告布局：左侧图表，右侧关键洞察：

```
图表区：x=40, y=120, w=700, h=480
洞察区：x=780, y=120, w=460, h=480
  - 核心结论（加粗，16px）
  - 3-5 个要点（14px）
  - 数据来源（12px，灰色）
```

> 当 `page_rhythm = breathing` 时，此标准布局可合理退化为**图表主导（2:8）**：一张大图表承载整页，一句要点概括——节奏规则由此满足，无需添加咨询风格外的模式。通用节奏纪律见 `executor-base.md §2.1`。

## 专业表达标准

### 页面标题

咨询风格页面标题应为**断言式标题**，非描述性标题：

| 类型 | 描述式（避免） | 断言式（推荐）                           |
| ---- | -------------- | ---------------------------------------- |
| 市场 | "市场概述"     | "国内市场同比增长 23%，显著跑赢全球平均" |
| 竞争 | "竞争分析"     | "三大竞争对手在渠道覆盖方面存在明显短板" |
| 财务 | "财务数据"     | "毛利率连续四季度改善，突破行业天花板"   |

### 数据来源标注

每个含数据的页面底部必须包含来源说明：

```xml
<text x="40" y="700" font-size="10" fill="#94A3B8">
  来源：国家统计局 2025 年度报告；内部团队分析
</text>
```

### 要点提示框

咨询风格内容页应在标题下方包含**要点提示框**（Takeaway Box）：

```
位置：x=40, y=80, w=1200, h=50
背景：浅主题色（fill-opacity="0.08"）
文字：14-16px，一句话概括该页核心要点
```

## 演讲备注风格

### 叙述基调

咨询风格演讲备注使用**结论先行**——先陈述结论，再呈现支撑证据。语调专业、简洁、有说服力。

### 备注撰写指南

备注为纯口述叙述（TTS）。不用方括号标记、不写 `要点：` / `时长：` 行——见 [executor-base.md §8](executor-base.md#8-演讲备注生成框架)。

- **结论先行**：每页备注的第一句即为核心要点。
- **口语化数据**：30% → "约三分之一"，85% → "十之八九"，2.5倍 → "两倍半"。口语形式更自然时将百分比拼写为文字。
- **证据紧随**：结论之后，以流畅的行文提供 2-3 个支撑数据点或事实。
- **自然过渡**：第一页后的每页以衔接上一页的句子开头（"基于上述扫描……"、"根据这些发现……"）。
- **专业用语**：使用"洞察"、"驱动因子"、"关键杠杆"等术语。

### 备注示例

```markdown
# 03_关键发现

基于我们的市场扫描，我们提炼出三个关键发现，其中最重要的是增长引擎正在从拉新转向留存。新客获取成本同比上升了近百分之四十，而改善的复购率贡献了超过百分之六十的营收增长。此外，下沉市场的增速显著超过一二线城市，二十五到三十五岁女性用户群的 ARPU 已接近整体平均的两倍。
```

## 自检补充（咨询型专属）

- [ ] 数据与源数据一致——不得编造
- [ ] 每个含数据的页面底部有来源标注
- [ ] 页面标题为断言式，非描述性
- [ ] KPI 卡片有趋势箭头和对比注释
- [ ] 图表配色统一，使用单色方案而非彩虹色
- [ ] 备注结论先行，数据口语化
- [ ] 备注不含方括号阶段标记和 `要点：` / `时长：` 元信息行（TTS 会逐字朗读）
