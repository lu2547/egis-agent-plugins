# 平安风格模板设计规范

> 适用于平安体系内外部汇报、金融保险专题、经营分析、智能体/AI 建设汇报、项目复盘、管理层汇报和数据型方案说明。整体应体现“稳健、可信、专业、结论清晰、信息充分”的金融机构表达方式。

---

## 一、模板总览

| 属性 | 说明 |
| --- | --- |
| 模板名称 | `pingan_style`（平安风格报告模板） |
| 适用场景 | 领导汇报、经营分析、保险金融专题、投资顾问汇报、组合回顾、市场点评、AI 平台建设汇报 |
| 设计基调 | 稳健正式、结论先行、信息密度适中偏高、强调口径清晰和数据可信 |
| 主题模式 | 浅色报告风：白色 / 浅灰背景 + 平安橙品牌强调 + 深灰正文 |
| 内容倾向 | 多用指标、表格、流程、对比、风险提示和实施计划，避免营销页式空泛表达 |

生成重点：平安风格不是“满屏橙色”，而是以平安橙作为品牌锚点，用大面积白底、灰色结构线、淡橙提示带承载正式金融报告气质。页面要有内容厚度，不能做成“少字大留白”的互联网演示风。

## 二、画布规范

| 属性 | 数值 / 要求 |
| --- | --- |
| 页面比例 | 标准 16:9 |
| 画布尺寸 | 1280 x 720 px |
| viewBox | `0 0 1280 720` |
| 页面边距 | 左右 72px，上方 72px，下方 32-60px |
| 安全区域 | x: 72-1208，y: 72-688 |
| 内容区建议 | 标题区 y: 72-122；提示区 y: 144-194；主内容区 y: 216-630；页脚 y: 660-688 |

页面排版优先使用稳定的横向网格：标题区、关键结论区、主体内容区、页脚来源区。除封面和结束页外，不建议使用大面积斜切、漂浮装饰和复杂渐变背景。

| 纯黑文字 | 正文不用 `#000000`，优先 `#333333` |

## 四、字体系统

### 1. 字体栈

正文和标题默认使用：

```text
Microsoft YaHei, PingFang SC, Arial, sans-serif
```

封面标题和结束页主标题可使用更正式的楷体栈：

```text
STKaiti, KaiTi, 华文楷体, Microsoft YaHei, PingFang SC, serif
```

### 2. 字号层级

| 层级 | 用途 | 字号 | 字重 |
| --- | --- | --- | --- |
| H1 | 封面主标题 | 44-56px | 700 |
| H2 | 内容页主标题 | 38-40px | 700 |
| H3 | 模块标题 / 卡片标题 | 24-28px | 700 |
| H4 | 表格标题 / 小节标题 | 18-22px | 700 |
| P | 正文段落 | 16-20px | 400 / 500 |
| Data | 关键数字 | 36-52px | 700 |
| Label | 指标说明 / 标签 | 12-16px | 500 / 700 |
| Note | 来源、页码、注释 | 12-14px | 400 |

### 3. 文本风格

- 标题要短、明确、结论化，避免口号式空话。
- `{{KEY_MESSAGE}}` 应写成一句可被领导快速理解的核心结论。
- 正文优先使用“结论 + 依据 + 动作/建议”的结构。
- 数据必须尽量附带口径、时间范围、来源或假设。

## 六、页面类型

### 1. 封面页：`01_cover.svg`

- 使用城市背景图作为上半部分视觉背景，表现集团、金融、城市服务和企业级场景。
- 中部使用大面积平安橙渐变标题面板，承载 `{{TITLE}}`、`{{SUBTITLE}}`、`{{AUTHOR}}`、`{{DATE}}`、`{{COVER_QUOTE}}`。
- 底部白色区域放置平安 logo 和 `{{BRAND_LABEL}}`。
- 标题可使用楷体栈，增强正式汇报和内部材料质感。
- 不额外添加卡通插画、科技光效、复杂装饰图形。

### 2. 章节/目录页：`02_chapter.svg`

- 本模板将目录页和章节页合并，不单独提供 TOC 页。
- 顶部放 `{{PAGE_TITLE}}` 和当前章节编号。
- 当前章节使用横向橙色高亮块展示 `{{CHAPTER_NUM}}`、`{{CHAPTER_TITLE}}`、`{{CHAPTER_DESC}}`。
- 下方用 2 列 x 3 行目录卡片展示 6 个章节项。
- 目录卡片保持白底灰边，编号使用柔和橙，文字用深灰。

### 3. 内容页：`03_content.svg`

- 页面背景为 `#FAFAFA`，主内容区域为白色面板。
- 顶部标题和分割线固定，形成统一报告页头。
- `{{KEY_MESSAGE}}` 必须放在主体内容上方，作为页面核心结论。
- `{{CONTENT_AREA}}` 是自由画布，生成器应根据内容选择表格、KPI、流程、矩阵、图表或左右分栏。
- 页脚必须保留 `{{SOURCE}}` 和页码，金融汇报场景要重视来源与口径。

### 4. 结束页：`04_ending.svg`

- 延续封面城市背景和橙色面板。
- 主体展示 `{{THANK_YOU}}`、`{{ENDING_SUBTITLE}}`、`{{CLOSING_MESSAGE}}`。
- 底部保留 logo、联系方式、日期和版权。
- 结束页保持克制，不做花哨感谢页。

## 七、内容页版式模式

| 模式 | 适用场景 | 生成要求 |
| --- | --- | --- |
| KPI 指标组 | 经营数据、项目成效、平台指标 | 2 x 2 或 2 x 3 网格，关键数字用橙色，附口径说明 |
| 明细表格 | 项目计划、任务清单、能力清单、风险清单 | 表头淡橙或极淡橙，文字深灰，行距紧凑但可读 |
| 左文右图 | 架构说明、方案解读、图表分析 | 左侧给结论和要点，右侧放图或图表，避免只有图无解释 |
| 对比矩阵 | 现状/目标、新旧版本、方案对比 | 横向列对比，橙色只突出重点差异 |
| 流程时间线 | 建设路径、业务流程、实施计划 | 阶段编号用橙色，节点说明保持短句 |
| 架构分层图 | 技术架构、能力架构、平台底座 | 按层次排布，使用灰线和淡橙节点，不要堆叠成流程图 |
| 风险与措施 | 项目风险、合规要求、运维保障 | 风险、影响、措施、责任人分列呈现 |
| 结论清单 | 管理层摘要、阶段总结 | 每条包含“结论 + 依据/动作”，不要只有标题词 |

## 八、内容生成要求

### 1. 领导汇报口径

- 每页先给结论，再给依据。
- 重点表达“做了什么、解决什么问题、当前进展、下一步计划、需要决策什么”。
- 避免大段技术细节堆砌，技术内容要转成能力、价值、风险和计划。
- 对 AI、智能体、RAG、PPT 生成等技术主题，要补充业务落点和建设阶段。

### 2. 金融报告口径

- 所有数字尽量带单位、周期、口径和来源。
- 风险、合规、权限、审计、可追溯等内容要有明确位置。
- 表格内容优先按“事项、现状、问题、措施、责任、状态”组织。
- 对比类页面要明确当前值、目标值、差异和后续动作。

### 3. 内容密度

- 一页内容不应只有 3 个词或 1 张孤立图。
- 每页至少包含一个明确的 `{{KEY_MESSAGE}}`。
- 图表页旁边或下方必须有结论解释。
- 架构页必须有分层、职责和调用关系，不能只画无说明方块。
- 项目计划页建议包含时间、任务、负责人、状态和风险。

### 4. 表格规范

| 项目 | 要求 |
| --- | --- |
| 表头 | 使用 `#FBD3BF` 或 `#FFF3EC`，不使用整行高饱和橙 |
| 表格线 | 使用 `#DCDCDC` 或 `#E2E2E2` |
| 正文 | 深灰 `#333333`，字号 13-16px |
| 强调 | 关键状态或数字用 `#F05A23`，但每行最多 1-2 个强调点 |
| 行高 | 信息密集但不拥挤，优先保证可读性 |

| 元素 | 建议值 |
| --- | --- |
| 页面左右边距 | 72px |
| 标题到关键结论区间距 | 22-32px |
| 关键结论区高度 | 48-56px |
| 主内容区顶部 | y=216 左右 |
| 模块间距 | 18-28px |
| 卡片间距 | 16-24px |
| 内容面板圆角 | 4px |
| 卡片圆角 | 4-6px |
| 表格内边距 | 8-14px |

## 十、SVG 技术约束

### 1. 必须遵守

| 规则 | 要求 |
| --- | --- |
| 画布 | `viewBox="0 0 1280 720"` |
| 背景 | 使用 `<rect>` 或 `<image>`，不要依赖 CSS 背景 |
| 文本换行 | 使用 `<tspan>`，不使用 `<foreignObject>` |
| 样式 | 使用内联属性，不使用 `<style>` 和 `class` |
| 透明度 | 使用 `fill-opacity` / `stroke-opacity` |
| 渐变 | 放在 `<defs>` 中，用 `<linearGradient>` 定义 |
| 图片 | 可使用内联 base64 或模板目录内稳定资产 |

### 2. 禁用元素

以下 SVG 特性不适合 PPT 转换，禁止使用：

- `clipPath`
- `mask`
- `<style>` 标签
- `class` 属性
- `foreignObject`
- `textPath`
- `animate*` 动画元素
- `script`
- `rgba()` 颜色格式

### 3. 阴影与质感

- 内容页不建议使用复杂 `filter` 阴影。
- 如需层次感，优先用浅灰边框、淡背景、轻微色阶区分。
- 封面和结束页可以使用图片背景和橙色面板形成层次，不再额外叠加复杂光效。

## 十二、质量检查清单

生成平安风格 PPT 时，逐页检查：

1. 是否使用平安橙作为品牌锚点，而不是整页橙色堆叠。
2. 是否每页都有清晰 `{{KEY_MESSAGE}}`。
3. 是否包含足够的指标、明细、表格、流程或依据。
4. 是否有数据来源、页码、口径说明。
5. 是否避免过度装饰、强阴影、大圆角和非品牌色大面板。
6. 是否符合金融报告的稳健、正式、可追溯表达。
7. 是否所有文本都在安全区内，且不会遮挡图表或边框。
8. 是否遵守 SVG/PPT 兼容约束。
